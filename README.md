# ⛩️ 広島・宮島 2泊3日 旅のしおり

家族旅行用の旅行プランサイトです。旅程タイムライン・地図（Leaflet + OpenStreetMap）・持ち物チェックリストを内包しています。

🔗 **サイト:** https://morinaoden.github.io/hiroshima/

## データ管理（Firebase編集モード）

旅程データの本体は **Firebase Firestore** です。サイト内の「編集」ボタンから予定の追加・変更・並べ替え・日またぎ移動・削除ができ、保存すると**家族全員の画面に数秒で自動反映**されます。

- 有効化の手順：[SETUP-FIREBASE.md](SETUP-FIREBASE.md)（約30分・カード登録不要）
- 接続設定：[js/firebase-config.js](js/firebase-config.js)（`null` のままだと従来どおり `data/itinerary.json` を表示し、編集モードは無効）
- `data/itinerary.json` は**初回シード＋接続失敗時のフォールバック**として残しています

## 主な機能

- 📅 日ごとのタイムライン（sticky タブで切替、日付・曜日表示）
- 🗺️ 地図との双方向連動（旅程タップ→地図移動、ピンタップ→旅程ハイライト）＋日ごとのルート線
- ▶ 旅行当日は「いまここ」「つぎ」を自動ハイライト、ヒーローに出発カウントダウン
- 📸 スポット写真＋手書き風メモ
- 🏷️ 「要予約」などの注意バッジ、イベント間の移動時間表示
- 📍 各スポットの「Googleマップで開く」リンク（タップでナビ起動）
- ✅ 持ち物チェックリスト（チェック状態・追加項目は端末に保存）
- 📱 モバイルでは地図がボトムシート（右下の「地図」ボタン）

## 旅程の編集方法

旅程はすべて [`data/itinerary.json`](data/itinerary.json) に入っています。このファイルを編集して push するだけでサイトに反映されます。

### 全体設定

```json
{
  "startDate": "2026-09-06",   // 1日目の日付。カウントダウン・いまここ表示に使用
  "packingList": ["着替え（3日分）", "..."]   // 持ち物リスト
}
```

### イベントの形式

```json
{
  "time": "11:30",
  "icon": "🥢",
  "title": "お好み焼きランチ",
  "description": "補足説明（空文字でもOK）",
  "spot": { "name": "お好み村", "lat": 34.3917, "lng": 132.4632 },
  "image": "https://…/photo.jpg",
  "caption": "鉄板アツアツ、本場の味！",
  "badge": "予約推奨",
  "travelAfter": { "icon": "🚗", "text": "車で約50分" }
}
```

| フィールド | 必須 | 説明 |
|---|---|---|
| `time` / `icon` / `title` | ✅ | 時刻・絵文字アイコン・タイトル |
| `description` | — | 補足説明 |
| `spot` | — | `null` でピンなし。緯度経度は Google マップで右クリックしてコピー |
| `image` | — | 写真URL（読み込み失敗時は自動で非表示） |
| `caption` | — | 写真の下の手書き風メモ |
| `badge` | — | 「要予約」などの注意バッジ |
| `travelAfter` | — | 次のイベントへの移動情報（隙間に表示） |

絵文字アイコンは `js/app.js` の `FLAT_ICONS` でフラットSVGに変換されます（未登録の絵文字はそのまま表示）。

## 写真クレジット

スポット写真は [Wikimedia Commons](https://commons.wikimedia.org/) のパブリックドメイン／CCライセンス画像を使用しています。ご自身の写真に差し替える場合は `image` のURLを変更してください。
